robots.txt

Robots.txt is a text document you add to your website which gives instructions about your site to roaming web robots, for example those employed by search engines. It tells them which parts of the website they can or cannot visit, but also who can access those parts, as certain robots can be granted special privileges. This is called the Robots Exclusion Protocol. Certain bots, such as malware robots might not follow the rules in robots.txt, and might even specifically look in the places they're told not too. The robot.txt file on this is page is configured so robots cant access the contacts page.

humans.txt

Humans.txt is an initiative started to get to know the authors of a website. In the text file there is information about who created the website and what their roles were. There is always information for robots, so why not for the humans? The humans.txt file for this website contains information about me, and some facts about the website, according to the standard.

Open Graph

Open Graph is a protocol that enables any web page to become an object in a social graph. mostly used by sites like Facebook and Twitter, it makes links to your website posted on social media look very aesthetically pleasing, while also enabling you to add other social media functionality like a Facebook-like button. you use it by adding meta tags to the head part of your HTML file.

The importance of separating layout and design

By Thomas Jorna

Separating layout [HTML] and design [CSS] is important for a variety of factors:

  • You're not repeating CSS elements.

    If you would use internal stylesheets you would have to define every paragraph to be a certain colour every time you would use the paragraph tag. If you use one external stylesheet, you just have to declare to every paragraph is red once.

  • Easier task management

    If you have different people working on design and layout, than it easier to separate the two. People concerned with the layout can focus on that, while the designers can work their magic in CSS

  • Readability

    Instead of having one humongous HTML file, you can have two smaller sized files where it is clear what is declared where.
